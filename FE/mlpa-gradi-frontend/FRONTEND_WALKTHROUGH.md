# 프론트엔드 워크스루 - Gradi (MLPA)

본 문서는 Gradi 프로젝트의 주요 프론트엔드 기능 및 아키텍처 구현 사항을 요약합니다.

## 🚀 주요 기능

### 1. 시험 이력 및 일괄 관리
- **관리 모드 전환**: 헤더의 보라색 그라데이션 톱니바퀴 아이콘을 통해 "조회 모드"와 "관리 모드"를 전환할 수 있습니다.
- **일괄 선택**: 관리 모드에서는 시험 항목 옆에 체크박스가 표시되어 여러 개의 시험을 한꺼번에 선택할 수 있습니다.
- **전체 선택**: 클릭 한 번으로 현재 검색된 모든 항목을 선택하거나 해제할 수 있는 기능을 제공합니다.
- **일괄 삭제**: 선택된 모든 시험 목록을 확인하고, 데이터 복구 불가능 안내와 함께 일괄 삭제를 진행하는 확인 모달을 구현했습니다.
- **상호작용 스타일**: 모든 클릭 가능한 요소에 `cursor-pointer`를 적용하여 사용자 경험을 개선했습니다.

### 2. 결과 및 통계 데이터
- **동적 학생 목록**: 시험 코드를 기반으로 실제 학생 데이터와 채점 결과를 실시간으로 조회하여 표시합니다.
- **예외 상태 처리**: 학생 데이터가 없는 경우 "열람할 수 있는 학생 데이터가 없습니다."라는 친절한 안내 메시지를 표시합니다.
- **다운로드 연동**: S3 Presigned URL을 통해 학생별 결과 파일 및 원본 출석 코드를 안전하게 다운로드할 수 있도록 연동했습니다.

### 3. 시스템 자동 유지관리
- **전역 자동 새로고침**: `AutoRefresh` 클라이언트 컴포넌트를 통해 5분마다 `window.location.reload()`를 실행하여 최신 데이터를 유지합니다.

### 4. 고급 업로드 오케스트레이션 (Saga 패턴)
- **시험 생성 프로세스**: [시험 생성 -> SSE 연결 -> 출석부 업로드 -> 이미지 배치 업로드]의 복합적인 과정을 하나의 흐름으로 관리합니다.
- **롤백 로직**: Saga 패턴을 적용하여 중간 단계 실패 시 이미 생성된 시험 정보를 자동 삭제하고 업로드된 파일을 정리하는 보상 트랜잭션을 구현했습니다.

## 🛠 아키텍처 및 기술 스택
- **프레임워크**: Next.js (App Router).
- **스타일링**: Tailwind CSS 기반 커스텀 보라색 그라데이션 및 애니메이션.
- **로직 관리**: 비즈니스 로직 분리를 위한 커스텀 훅(`useExamForm`, `useExamHistory`) 사용.
- **API 통신**: Axios/Fetch를 활용한 중앙 집중형 `examService` 구현.
